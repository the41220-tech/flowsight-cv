# FlowSight 대시보드 기획서 (초안 v0.1)

**작성:** 2026-06-22 · **상태:** v0.2 — **MVP 범위 확정: 흐름+밀도+안전** (추적기 제외, §12) · **원칙:** *안전은 하나의 렌즈, 분석이 척추다.*

> FlowSight = 군중 **움직임 분석·해석** 플랫폼. 대시보드도 "압사 경보판"이 아니라
> **크라우드 무브먼트 인텔리전스**로 설계한다. 분석(흐름·밀도·동선·추세)이 일상 사용이고,
> 안전 경보는 그 위에 얹힌 예외 레이어다.

---

## 0. 한 줄 정의
드론/CCTV 영상 → 사람 위치·밀도·흐름장 복원 → **군중이 어떻게 움직이는지 정량 분석**하고,
이상 동역학은 경보로 surface 하는 단일 대시보드.

## 1. 포지셔닝 — 왜 "분석 중심"인가
- 안전 경보는 **저빈도·고임팩트**(평소 0회). 그것만으론 대시보드를 매일 열 이유가 없다.
- 분석(흐름 효율, 혼잡, 동선, 체류, 시간대 패턴)은 **고빈도·상시 가치** → 운영·기획·마케팅이 매일 본다.
- 같은 흐름장/밀도장에서 안전 신호도 파생되므로, **분석을 깔면 안전은 공짜로 따라온다**(역은 성립 안 함).
- 경쟁사 평면 히트맵과의 차별점도 분석 깊이(비평면 3D 측위 + 방향성 흐름)에서 나온다.

## 2. 타깃 사용자 & 핵심 질문(Jobs-to-be-done)

| 사용자 | 모드 | 핵심 질문 |
|---|---|---|
| 운영/관제 | 실시간 | 지금 어디가 막히나? 흐름이 정상인가? |
| 기획/분석가 | 사후·추세 | 어느 구역·시간대가 붐비나? 병목은 어디고 왜인가? |
| 경영/마케팅 | 의사결정 | 사람이 어디 모이고 머무나(핫스팟·체류)? 동선은? |
| 안전책임자 | 예외 | 이상 징후(압사·패닉·서지·역류·낙상)가 있나? |

→ 대시보드는 위 4개 질문에 **하나의 화면 흐름**으로 답한다. 안전은 마지막 한 줄.

## 3. 분석 축 5개 (안전은 5번째)

1. **흐름 분석(Flow)** — 속도장·방향·**divergence(확산/이탈)·curl(회전/와류)·counterflow(역류)·convergence(쏠림)**. 흐름 효율, 충돌·정체 구조.
2. **밀도·점유(Density/Occupancy)** — 시공간 밀도맵, 점유율, capacity 대비, 시간대별 추이.
3. **동선·체류(Movement/Dwell)** *(운영+마케팅)* — path, dwell time, 진입/이탈, **핫스팟**, OD(origin–destination).
4. **추세·비교(Trend)** — 시간대/요일/이벤트 전후 비교, **정상 대비 이탈** 정량화.
5. **안전·이상(Safety/Anomaly)** — 이상 동역학 경보 + 3D 점군. *(분석 위에 얹힌 한 층)*

## 4. 정보 구조 (탭/뷰)

```
┌ Overview ─ 현재 인원·밀도·흐름효율·이상상태 KPI 한눈에
├ Flow Analytics ─ 속도/방향장, 역류·확산 히트맵, 흐름효율 시계열
├ Density & Occupancy ─ 시공간 밀도맵, 점유율 게이지, 시간대 추이
├ Movement & Dwell ─ 동선 히트맵, 체류시간 분포, 핫스팟, OD (★추적 필요)
├ Trend & Compare ─ 기간·시간대 비교, 정상 baseline 대비
└ Safety / Anomaly ─ 이상 타임라인, 경보, 3D 점군 (실영상 결과)
```

## 5. 패널별 상세 (지표 · 데이터소스 · 가용성)

| 패널 | 핵심 지표 | 데이터 소스(현 모듈) | 지금 가능? |
|---|---|---|---|
| Overview | 인원수, 평균밀도, 흐름효율, 이상상태 | detect, density, flow | ✅ |
| Flow | 속도장, divergence/curl/counterflow, 흐름효율 | dense optical flow | ⚠️ **방향성 특징 레이어 신규 필요**(작음) |

| Movement/Dwell | path, dwell median, OD, 핫스팟 | **멀티오브젝트 추적 필요** | ❌ 미구현(추적기 추가 필요) |
| Trend | 시간대·기간 비교, 정상 대비 | 위 지표의 시계열 누적 | ⚠️ 로깅/저장 레이어 필요 |
| Safety/Anomaly | 이상 타임라인, 경보, latency, 3D 점군 | `disaster_v1`, rvt 파이프라인 | ✅ (단 경보 precision 정제 필요) |

## 6. 지표 사전 (초안)
- **흐름효율 Flow Efficiency** = 주방향 정렬 속도 / 전체 속도(0~1). 낮을수록 충돌·정체.
- **역류비 Counterflow Ratio** = 주류 반대방향 이동 픽셀 비율.
- **확산/쏠림** = ∇·v(divergence) 부호·크기 (양=확산/이탈, 음=쏠림/수렴).
- **와류 Curl** = ∇×v 크기 (군중 회전·소용돌이 = 위험 전조).
- **점유율 Occupancy** = 현재밀도 / 설계 capacity(%).
- **체류 Dwell(향후)** = 트랙별 체류 시간 중앙값.
- **이상 리드 Anomaly Lead** = onset − alarm (선행초). **정밀도 Precision/Recall** = 경보 신뢰도.

## 7. 데이터 파이프라인 매핑 (모듈 → 지표)

```
영상 ─▶ 검출 FT-2(+SAHI) ─▶ 카운트·밀도장 ─▶ [점유·혼잡 분석]
                       └▶ 3D 측위(H1) ─▶ 지면좌표 ─▶ [공간 분석·히트맵]
영상 ─▶ dense optical flow ─▶ 속도장 ─▶ (방향성 특징 레이어) ─▶ [흐름 분석]
                                                              └▶ [안전 이상 경보]
   (※ 추적기 추가 시) ─▶ 트랙 ─▶ [동선·체류·OD = 마케팅 분석]
```

## 8. MVP vs 향후 (정직한 갭)

**MVP (지금 데이터로 가능):**
- Overview + Density/Occupancy + Safety/Anomaly(실영상 결과 기반) → 즉시.
- Flow Analytics → **방향성 흐름특징 레이어 1개**만 추가하면 가능(데이터 0).

**향후 (신규 빌드/데이터 필요):**
- **Movement/Dwell/OD/마케팅 분석** → 멀티오브젝트 **추적기**(ByteTrack/BoT-SORT 등) 필요. ← 분석 깊이의 최대 게이트.
- Trend/Compare → 지표 시계열 **로깅·저장** 레이어 필요.
- 실시간 라이브 → 스트리밍 + 추론 최적화(엣지).

> **핵심 시사점:** "안전 외 분석"(특히 마케팅·동선)을 제대로 하려면 **추적기**가 분기점이다.
> 흐름특징 레이어(흐름 분석)와 추적기(동선·체류 분석)가 분석 대시보드의 두 핵심 빌드.

## 10. 대시보드 자체 성공지표 & 리스크
- 성공: "매일 여는 분석 도구"가 되는가(안전 경보 없는 날에도 가치). 핵심 질문 4개 답하는가.
- 리스크: ① 분석 지표 신뢰도(검출 recall·경보 precision 현재 한계) → 수치에 신뢰구간/한계 명시. ② 추적 부재로 마케팅 분석이 얕음 → MVP에서 과약속 금지.

## 12. 확정 MVP 사양 (2026-06-22 결정: **흐름 + 밀도 + 안전**, 추적기 제외)

> §11-Q1 결정됨 → (a) 안전+밀도+**흐름 분석**. 동선·체류·OD·마케팅(추적기 필요)은 **2차로 분리**.
> 이로써 "분석 중심" 취지를 *지금 가능한 최대 깊이*(방향성 흐름 + 밀도/점유)로 구현한다.

### 12.1 빌드 ① — 방향성 흐름특징 레이어 (신규 `flowsight/physics/flow_features.py`)
dense optical flow(이미 rvt가 계산)에서 **셀 단위** 산출:
- 평균 속도장 `(vx, vy)`, `speed=|v|`
- **divergence** `∇·v` — 양=확산/이탈, 음=쏠림/수렴
- **curl** `∇×v` — 와류(군중 회전 = 위험·혼잡 전조)
- **counterflow ratio** — 주류 반대방향 이동 비율
- **flow efficiency** — 주방향 정렬 속도 / 전체 속도 (0~1)
- **speed-shock** — `d(speed)/dt` (패닉/서지 onset)
→ 프레임/구역별 **시계열 + 맵** 저장(`flow_features.json` + 맵 PNG). 신규 데이터 0(UMN 재사용).

### 12.2 데이터 런 (Colab)
캐시된 `rvt_arrays.npz` 흐름성분 재사용 가능 여부 점검 → 부족 시 dense flow 재산출(1런). 산출물 Drive 저장.

### 12.3 대시보드 패널 (Overview + 3)
- **Overview KPI** — 인원 · 평균밀도 · **흐름효율** · 이상상태.
- **Flow Analytics** — 속도/방향 화살표장 + divergence/curl/counterflow **히트맵**, 흐름효율·역류비 **시계열**.
- **Density & Occupancy** — 밀도 히트맵, peak/avg, 점유율, 시간대 추이.
- **Safety / Anomaly** — 이상 타임라인(7 에피소드, 경보 vs onset), 3D 점군, latency + **precision 한계 명시**.

### 12.4 빌드 순서
1. `flow_features.py` 작성 + 로컬 단위검증 →
2. Colab 배포·런 → `flow_features.json` →
3. 자가완결 HTML 대시보드(실데이터, 오프라인) →
4. 렌더 검증 + repo 커밋.

### 12.5 범위 밖 (2차)
추적기(ByteTrack/BoT-SORT) 기반 **동선·체류·OD·마케팅 분석**, 실시간 라이브, 추세 로깅·비교.
