# FlowSight — 로드맵 반복 루프 (fugu 주도 · Claude 실행)

남은 로드맵(D-VLM, E 멀티카메라 하드닝, F 외적검증)을 **검증 → 수정 → 보고** 사이클로 전진시키는 절차.

## 역할
- **fugu (Sakana)** — 다음 단계 *제안* + 합격 기준 정의, 결과 *리뷰* + 사이클 보고서. (LLM, 도구 없음)
- **GLM-5.2** — (선택) 성과/문서 요약.
- **Claude** — *구현·수정·검증*(코드/테스트/진단 실행). **도구를 가진 유일한 실행 주체.**

## 한 사이클의 단계
1. **제안 (fugu)** — 백로그에서 다음 항목 1개 + 수용 기준(통과해야 할 테스트/지표) 제시.
2. **구현·검증 (Claude)** — 코드 작성/수정 → `tests/` 전체 통과 + 필요한 진단 실행. **테스트 실패 시 사이클 미완료.**
3. **리뷰·보고 (fugu)** — 결과를 합격 기준 대비 평가 + 사이클 보고서 작성.
4. **기록** — `docs/`에 사이클 로그, `MEMORY.md` 갱신, (코드 변경 시) 사용자 Mac에서 push.

## 트리거 (중요 — 자율 무한루프 아님)
`ask_model`(fugu/GLM)은 **명시적 사용 전용**(자율·자기검증 호출 금지)이 계약이다. 따라서 본 루프는 **사용자가 사이클을 트리거**할 때마다 1회 전진한다:
- 사용자: **"다음 사이클"** (또는 대상 지정, 예: "E의 교차뷰 연관") → Claude가 위 4단계를 1회 수행.
- 무인 백그라운드/스케줄로 fugu를 반복 호출하지 않는다(계약 위반 + 무감독 코드수정 위험 회피). 코드 수정은 항상 테스트 검증을 동반.

## 백로그 (우선순위)
1. **E — 멀티카메라 하드닝(차기 핵심 해자)**: ~~① 근지평선 클램프를 `wildtrack_validate.py`에 연결~~ ✅, ② 7뷰 전체 정량 평가(MODA/MODP), ③ 교차뷰 트랙 시간연관, ④ 학습형 멀티뷰 검출기(MVDet 계열)로 recall↑ — **실험 프로그램 준비됨**(`docs/RECALL_EXPERIMENT_PROGRAM.md`, `experiments/recall_lab.py`: 10가설 Registry + 동일 FPPI 비교 loop; cached tier 즉시, detector/train tier는 Colab).
2. **F — 외적검증·경보 정밀도**: WILDTRACK 정량(MODA/MODP), `HysteresisEventGate`를 anomaly/absolute/terror 러너에 연결, FT-3(CCTV recall).
3. **D — VLM 설명**: `narrate_vlm`(Qwen2.5-VL) 실연결.
4. 객체ID/개방어휘, 엣지 배포(v3).

## 사이클 로그
- **Cycle 1 (2026-06-23, 완료)** — *제안*: 실데이터에서 입증된 근지평선 투영 발산 → 클램프 필요. *수정*: `geometry/wildtrack.py`의 `to_ground(uv, bounds=...)`에 근지평선/경계 클램프 추가(기존 무인자 경로는 불변=하위호환). *검증*: `tests/test_wildtrack.py`에 클램프 테스트 추가 → **전체 27/27 통과**(moat2 12 + anomaly 10 + wildtrack 5). *다음*: 클램프를 `wildtrack_validate.py`에 연결(백로그 E①).
- **Cycle 2 (2026-06-23, 완료)** — *제안(fugu)*: bounds를 validation 경로 전체에 전달 + `GROUND_BOUNDS=(-3.0,-0.9,9.0,35.1)` 상수화. *수정*: `geometry/multicam.py` `CameraView.to_world(bounds=None)` + `MultiCameraFusion.fuse(bounds=None)` 시그니처 확장; `experiments/wildtrack_validate.py` `GROUND_BOUNDS` 상수 + 두 호출(single-cam, fuse)에 전달; `tests/test_moat2.py` `_IdentityCal` 하위호환 수정. *검증*: `tests/test_wildtrack.py` 신규 2개 추가(bounds 연동 + fuse bounds) → **전체 29/29 통과**(moat2 12 + anomaly 10 + wildtrack 7). *fugu 리뷰*: 모든 수용 기준 충족, 하위호환성 양호. *다음*: E② 7뷰 전체 정량 평가(MODA/MODP).
- **Cycle 3 (2026-06-23, 완료)** — *목표*: ~10% recall을 7캠+SAHI로 개선. *실험*: 4캠(C1,C2,C4,C5) + SAHI(slice=512, overlap=0.2, CONF=0.4), 20 frames, GT=637. *결과*: `[4cam+SAHI] prec=0.046 recall=0.089 fp=1195` vs `baseline 2cam=0.102` → **SAHI가 recall을 오히려 낮춤.** *근본원인*: (1) FT-2 드론→지상 도메인 불일치로 FP 폭발(21배), (2) CVLab1≈CVLab4 거의 동일 위치로 커버리지 무증가, (3) C3/C6/C7 zip 손상 복구 불가. *결론*: recall 문제는 geometry가 아닌 detector 도메인 문제. *필요 조건*: 완전한 7카메라 (WILDTRACK 재다운로드) + 도메인 적합 검출기(MVDet 계열) → E④ 학습형 멀티뷰 검출기 과제. *다음 트리거 시*: E② 7뷰 MODA/MODP (완전한 데이터 취득 선결).
- **Cycle 4 (2026-06-23, 완료)** — *목표*: recall ~0.1(=검출기 도메인 문제)을 체계적으로 푸는 **실험 프로그램** 설계·구축. *설계(fugu)*: Registry/Runner/Metrics/Comparator/Loop 아키텍처 + 동일 FPPI 비교 규칙(fugu는 긴 프롬프트 2회 타임아웃, 짧은 프롬프트로 설계 반환). *구현(Claude)*: `flowsight/eval/slice_metrics.py`(recall@IoU·AR·MR-2·FPPI/FROC·슬라이스별·**동일 FPPI 비교**), `flowsight/eval/nms_variants.py`(hard/Soft/DIoU/WBF, 재학습0), `experiments/recall_lab.py`(10가설 Registry + 비용순 loop + 합성 데모), 문서 `docs/RECALL_EXPERIMENT_PROGRAM.md`. *검증*: `tests/test_recall_lab.py` 8개 추가 → **전체 37/37 통과**(moat2 12 + anomaly 10 + wildtrack 7 + recall_lab 8). 데모: baseline recall 0.527; cached-tier(NMS/threshold) ΔRecall@FPPI +0.002~0.006 → REJECT(=값싼 추론옵션으론 도메인 문제 못 품을 재확인), detector/train tier는 Colab 런 스펙으로 연기. *다음 트리거 시*: detector-tier 실행(실 검출기로 H5 tiling/H10 pose) 또는 train-tier(H8 loss/H9 aug) — Colab + 실데이터.
- **Cycle 5 (2026-06-23, 완료)** — *제안(fugu)*: 후보 (a) 검출기 주입형 tiling+WBF 모듈 선택 + 수용기준 3개(①tile→remap→WBF 좌표복원 IoU≥0.99 ②크로스타일 동일박스 WBF 1개 병합 ③whole-frame recall 0→tiled recall>0, ΔRecall@matched-FPPI>0). *구현(Claude)*: `flowsight/eval/tiling.py`(`tile_grid` 겹침 타일, `run_tiled(detect 주입)` 타일별 추론→글로벌 remap→WBF, `to_local` 실어댑터 헬퍼). 검출기를 콜러블로 주입 → 임의 검출기(인도메인/head/pose)를 lab Comparator로 A/B 가능. *검증*: `tests/test_recall_lab.py`에 수용기준 3개 테스트 추가 → recall_lab 11/11, **전체 40/40 통과**(moat2 12 + anomaly 10 + wildtrack 7 + recall_lab 11). *fugu 리뷰*: **채택/머지 권장**(수용기준 충족, 40/40). *다음 트리거 시*: detector-tier **실데이터 eval** — 고정(인도메인) 검출기로 실 val에서 whole vs tiled를 동일 FPPI 비교(Colab + 클린 WILDTRACK 7뷰).
- **Cycle 6 (2026-06-23, 완료)** — *상황*: 기록된 다음(detector-tier 실데이터 eval)은 외부 의존(클린 7뷰 손상 C3/C6/C7 + 인도메인 검출기 부재 + Colab 불안정)으로 샌드박스서 닫을 수 없음. *제안(fugu)*: 강행 대신 **샌드박스 완결 가능한 occlusion 레버** 채택 — head/상체 검출(주입형)→기하 prior로 발 지면앵커 복원(H1/H10 추론측; tiling이 못 잡는 발-가림 회복) + 수용기준 3개. *구현(Claude)*: `flowsight/eval/body_prior.py`(`head_to_body`/`head_to_foot` 기하 prior: body_h=k·head_h, body_w=w_ratio·head_w; `merge_head_proposals` 주입형 head→body 제안 병합 + dedup + empty 견고). 실 head 검출기는 tiling과 동일 주입 패턴으로 Colab에서 연결. *검증*: 테스트 3개(①head→body IoU≥0.9 & foot k-단조 ②heads로 발-가림 회복 ΔRecall@matched-FPPI>0 ③empty/콜러블/dedup 견고) → recall_lab 14/14, **전체 43/43 통과**. *fugu 리뷰*: **채택**. *다음 트리거 시*: `merge_head_proposals`를 실평가 파이프라인에 기본 연결 + matched-FPPI 리그레션 체크(실 head 검출기/데이터 = Colab).
- **Cycle 7 (2026-06-23, 완료) — detector-tier 실데이터 eval (Colab L4).** *준비(Claude)*: `experiments/recall_realdata.py`(WILDTRACK 2D GT view별 파서 + 슬라이스태깅 + YOLO whole/tiled 어댑터 + recall-lab 동일FPPI 비교), mock으로 샌드박스 검증(파서/eval_view) → 전체 **44/44**. *Colab 배포*: 런타임 생존(WT.zip+추출본 C1–C5 보존, L4) 확인 → eval 5모듈+러너를 **tar+gzip+base64 1셀**로 레포에 해제(DEPLOYED). *실행*: `recall_realdata.py --view C1 --n 20`(yolo11x COCO, GT 534). *결과*: **[whole] recall@.5 0.567 / [tiled] 0.579**; 슬라이스 occluded **0.484→0.519(+0.035, tiling이 가림 약간 도움)**; **그러나 동일 FPPI: tiled Δrecall −0.024~−0.069(악화)** — 타일 경계 FP가 FP예산을 소모. *핵심 통찰*: 이전 "BEV 엔드투엔드 recall ~10%"와 달리 **순수 2D 검출 recall은 0.567** → 손실의 대부분은 검출이 아니라 발끝→지면 투영·융합 단계. *fugu 리뷰*: **tiling 미채택**(동일 FP예산서 손해). *다음*: WILDTRACK/C1 **도메인 적합 fine-tune 검출기** 학습 후 동일 FPPI 재평가(+ head 검출기로 `merge_head_proposals` 실연결).
- **Cycle 8 (2026-06-23, 완료) — 지면앵커 실험 (Colab L4, 사용자 지시: 발끝 고정 탈피+데이터보정+검증).** *구현(Claude)*: `flowsight/eval/anchor_proj.py`(bbox 세로분율 α 앵커 + 실캘리브 to_ground 투영 + `calibrate_alpha` 데이터 fit=fine-tune + occlude/head-extrapolate 헬퍼), `experiments/anchor_lab.py`(WILDTRACK view별 GT 2D박스↔positionID 월드 페어 + calib 자동선택 + train→test α fit/평가). 합성카메라 단위검증(발끝 가시→foot 정확/가림→보정α 우수) → 전체 **46/46**. 수정 wildtrack.py(구버전 우회)+2모듈 tar+base64 1셀 배포. *실데이터 결과(C1, GT박스, auto calib=CVLab1, train→test)*: **foot(α1.0) loc_err 7.97m / center(0.5) 3.27m / 데이터보정 α=0.625 → 2.18m(occluded 1.58m)** = 발끝앵커 대비 전체 **−73%**·가림 **−80%**(수용기준 15%/25% 대폭 초과). 핵심: **bbox 하단 ≠ 지면접촉점** — Cycle 7이 가리킨 투영 병목의 정체. 단 naive head-extrapolation은 7.97m로 폐기(정답은 "머리"가 아니라 **데이터보정된 수직 분율 α**). *fugu 리뷰*: **조건부 채택**(α 보정앵커 채택, foot/naive-head 폐기). 또한 auto-calib이 C1→**CVLab1** 선택 → Cycle 3의 "C1→CVLab3"는 발끝-노이즈였음 확인. *다음*: α를 wildtrack_validate/multicam 투영에 per-camera로 적용(운영화) → 검출기 박스로 BEV recall 재측정.
- **Cycle 9 (2026-06-23, 완료) — 보정앵커 BEV 운영화 (Colab L4, 실검출기).** *구현(Claude)*: `experiments/recall_bev.py`(per-camera α를 train프레임 GT로 fit → test프레임 실검출기(yolo11x) 박스에 적용 → `project_anchor`→`match_to_gt` BEV recall@1m/2m, foot vs 보정 비교; `bev_recall` 코어는 mock 단위검증). tests 18 → 전체 **47/47**. recall_bev.py + experiments/__init__.py tar+base64 배포. *실데이터 결과(C1, yolo11x, train20→test20, α*=0.625 fit_err 2.48m)*: **@1m foot 0.085→calib 0.125 (Δ+0.039, +46%), @2m foot 0.197→calib 0.318 (Δ+0.121, +61% rel, +12.1%p)** — 수용기준 ②(+5%p) 대폭 초과. 즉 **Cycle 8 보정앵커가 실검출기 엔드투엔드 BEV recall을 유의하게 향상**(사용자 지시 "지면투영 정확도 개선" 실적용 완료). 절대 recall은 여전히 단일 oblique뷰+COCO 검출기 한계로 modest지만, 보정앵커·멀티캠융합(Cycle7 2캠>1캠)·도메인검출기는 곱셈적. *fugu 리뷰*: **채택 ✓**. *다음*: α* 일반화 검증(C2~ 동일 train→test 프로토콜) → 멀티캠 융합에 보정앵커 결합.
