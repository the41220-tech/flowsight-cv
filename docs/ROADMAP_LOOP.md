# FlowSight — 로드맵 반복 루프 (fugu 주도 · Claude 실행)

남은 로드맵(D-VLM, E 멀티카메라 하드닝, F 외적검증)을 **검증 → 수정 → 보고** 사이클로 전진시키는 절차.

## 역할
- **fugu (Sakana)** — 다음 단계 *제안* + 합격 기준 정의, 결과 *리뷰* + 사이클 보고서. (LLM, 도구 없음)
- **GLM-5.2** — (선택) 성과/문서 요약.
- **Claude** — *구현·수정·검증*(코드/테스트/진단 실행). **도구를 가진 유일한 실행 주체.**

## 한 사이클의 단계
1. **제안 (fugu)** — 백로그에서 다음 항목 1개 + 수용 기준(통과해야 할 테스트/지표) 제시.
2. **구현·검증 (Claude)** — 코드 작성/수정 → `tests/` 전체 통과 + 필요한 진단 실행. **테스트 실패 시 사이클 미완료.**
3. **리뷰·보고 (fugu)** — 결과를 합격 기준 대비 평가 + 사이클 보고서 작성.
4. **기록** — `docs/`에 사이클 로그, `MEMORY.md` 갱신, (코드 변경 시) 사용자 Mac에서 push.

## 트리거 (중요 — 자율 무한루프 아님)
`ask_model`(fugu/GLM)은 **명시적 사용 전용**(자율·자기검증 호출 금지)이 계약이다. 따라서 본 루프는 **사용자가 사이클을 트리거**할 때마다 1회 전진한다:
- 사용자: **"다음 사이클"** (또는 대상 지정, 예: "E의 교차뷰 연관") → Claude가 위 4단계를 1회 수행.
- 무인 백그라운드/스케줄로 fugu를 반복 호출하지 않는다(계약 위반 + 무감독 코드수정 위험 회피). 코드 수정은 항상 테스트 검증을 동반.

## 5가설 등록부 (Cycle 10, 멀티에이전트 패널 산출 — 차례대로 검증)
패널(Explore 내부검증 + Plan 아키텍처비판 + general-purpose 최신연구 + GLM/fugu 외부)이 **독립 수렴**한 레버를 검증가능 가설로 정리. 순서 = 검증 가능성·레버 크기.
- **H1 — 높이-prior 머리/수직축 앵커가 전역 보정 α를 능가** (지면측위). *지표*: BEV loc_err·recall@1/2m. *합격*: 이질적 발-가림에서 머리앵커가 보정 α 대비 loc_err ≥25%↓, 보정데이터 0. *근거*: Zhang&Ye 2024(head>ankle), Niu 2021(수직세그먼트 <1m). → **검증 완료(Cycle 10): 합성 −75.5%(0.748→0.183m)**.
- **H2 — 학습형 멀티뷰 BEV 검출기(MVDet/3DROM/MvCHM 계열)가 단일뷰+후기융합 recall 천장 돌파** (검출). *지표*: 동일 FPPI MODA/MODP·recall@2m. *합격*: MODA ≥0.80(현 엔드투엔드 0.1–0.3 대비). *근거*: MVDet 88.2→3DROM 93.5(+5.3), MvCHM standing-point 제거 시 95.3→42.1. *블록*: 클린 7뷰(C3/C6/C7 재취득)+GPU 학습.
- **H3 — 불확실성 가중·지평선 인지 융합이 카메라 1→7 증설 시 recall 단조 증가(정밀도 붕괴 없이)** (멀티캠). *지표*: N캠별 prec/recall@1m(동일 FPPI). *합격*: 가중융합 7캠서 prec≥0.7 유지하며 recall↑; 무가중은 실패. *근거*: 현 `multicam.fuse` 무가중 평균+고정반경; MonoLoco 불확실성(단안 ~5%/거리 한계). → **검증 완료(Cycle 11): recall 0.5→1.0, precision 0.769→1.0, loc_err −30%**.
- **H4 — 속도-무질서(엔트로피/질서모수) 채널이 Helbing 압력보다 빠른 선행경보** (조기경보). *지표*: 실/UMN footage서 압력경보 대비 선행시간(s)·정밀도(히스테리시스 게이트 후). *합격*: 선행 중앙값 >0 & 이벤트 정밀도 ≥0.5. *근거*: 이태원 PLOS 2024(엔트로피 10.99, 압력피크 前 상승). → **검증(Cycle 11): 밀도-독립 Var(v)가 P 대비 +2.4s 선행 ✓; 방향-엔트로피는 coherent 시뮬서 ~0 → 난류/실footage 필요**.
- **H5 — 지형 위치에너지 채널은 진짜 비평면에서만 선행 부가가치(평지선 ≈0)** (해자 진실성). *지표*: 경사 실scene 선행(s) vs 평지 실scene 기여. *합격*: 경사서 ≥3s 선행 & 평지서 |기여|<잡음. *블록*: 실 경사-군중 footage(현 모든 지형결과 합성). *주의*: 해자의 핵심 차별점이 100% 합성검증 — 최우선 외적검증 대상. → **검증(Cycle 11): 평지 기여 0.0000 확인 ✓(구조 차별점 진짜); 경사 선행은 regime 의존(−0.8~+0.6s) 합격선 미달 → 실 경사footage 필요**.

**패널 disagreement 해소**: Plan이 "BEV recall 분모에 단일뷰 비가시 인원 포함(시야 confound)" 제기 → 코드 확인 결과 `anchor_lab.load_pairs`가 이미 `viewNum & xmax!=-1`로 **per-view 가시 인원만** 포함(분모 공정) → Plan #6 부분 반증. "geometry 0.0000m는 해석적 역함수라 거의 항등(실 정확도 보증 아님)"은 타당 → H1/H5가 실 정확도를 직접 겨냥. 3개 서브에이전트가 **앵커(H1)와 학습형 멀티뷰(H2)를 최상위 레버로 독립 수렴**.
