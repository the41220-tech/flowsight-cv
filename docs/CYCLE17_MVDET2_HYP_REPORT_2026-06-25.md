# Cycle 17 — MVDet 4-view 한계 돌파 실험 보고서 (5가설)

날짜 2026-06-25 · 데이터 WILDTRACK 4뷰(C1,C2,C4,C5) 40프레임(20 train/20 test) · T4 · BEV occupancy + peak-NMS(1.0m) · @2m 매칭 · 지표 = thr 스윕 best-F1

## 0. 가설 도출 절차 (Opus → fugu 비판 → 5 확정)
Opus가 한계 돌파용 10여 개 중 5개를 제안(H1 해상도↑, H2 깊은특징, H3 σ축소, H4 증강, H5 TTA).
fugu(Sakana) 비판 검토: **H2·H4 통과**, H1·H3·H5 탈락 + 대체안 R1/R2/R3 제시.
- H1 탈락: 4× 픽셀은 이득 적고 지연↑·20프레임 과적합.
- H3 탈락: σ0.3m는 캘리브 노이즈 이하, focal 튜닝 과적합.
- H5 탈락: TTA 지연↑ + ±1프레임은 **미래정보 누설**(실시간 경보 부적합) + hflip이 캘리브 멀티뷰 기하 파괴.
Opus 재확인 후 최종 5 = **HA=H2(깊은특징), HB=H4(증강), HC=R1(freeze+WD), HD=R2(캘리브 정렬), HE=R3(train-only thr 선택)**.

## 2. 해석 (CV 엔지니어 + 사업 관점)
- **HA 깊은특징(ResNet layer3 추가) = 유일한 천장 상승**: best-F1 0.471→0.492, thr0.05 precision 0.450→0.511, fp 213→160. 풍부한 의미정보가 BEV 오탐을 줄임. thr0.10에서 P1.000(무-오경보) 유지. **사업적**: 모델 약간 무거워지나 4캠 규모에서 배포 가능 + precision 직접 상승(트래킹/상용 분석 가치).
- **HB 증강 기각**: 20프레임 + per-view dropout이 오히려 BEV를 잡음화(fp 213→682). 데이터가 너무 적어 증강이 과적합 방지보다 노이즈를 더함.
- **HC freeze+WD 기각(최대 하락)**: 백본을 얼리면 ImageNet 특징만 남아 광장 도메인에서 변별력↓(fp 2991). **fine-tuning이 정규화보다 중요**. (단 thr0.02 recall 0.835로 순수 recall은 최고 → 압사경보 한정 동작점으로는 의미 있음.)
- **HD 캘리브 정렬 중립**: 5프레임 probe가 이미 안정 매핑({C1:CVLab1,C2:CVLab2,C4:IDIAP3,C5:CVLab2})을 찾아, 전프레임으로 넓혀도 동일 → 이 고정 4뷰 세트에선 캘리브가 병목이 아님. (fugu의 "캘리브가 4캠 recall 지배"는 일반론으론 맞으나 여기선 이미 해결됨.)
- **HE train-thr 채택(방법론)**: TRAIN에서 고른 동작점(thr0.07,nms0.5)을 TEST에 적용 → F1 0.418, oracle(테스트 엿보기) 0.436과 −0.018 차 → **thr/NMS는 test 누설 없이 train에서 정직하게 골라도 거의 손실 없음**. 보고 프로토콜로 채택.

## 3. 핵심 결론 (가장 중요)
**20 train 프레임 regime은 run-to-run 분산이 커서(예: cvthr run의 thr0.05 R0.636 vs baseline R0.494) 아키텍처 미세조정 5개 중 3개가 오히려 퇴보**했다. 이는 fugu가 지적한 과적합 위험을 실증한다. → **진짜 천장은 데이터 양**이다. 단일 명확한 상승(HA 깊은특징, +0.021)도 분산 내일 수 있어 시드 반복 확인이 필요.
